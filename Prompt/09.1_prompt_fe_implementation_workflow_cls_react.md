# Full Implementation Workflow Prompts — CLS Frontend (React + Vite + JavaScript)

_Bộ lệnh Prompt dành cho AI (AI Coding Formula) giúp xây dựng Frontend của **Classroom Management System (CLS)** tuân thủ cực kỳ chuẩn xác các quy tắc được định nghĩa trong `coding-conventions.md`._

**Tech Stack xác định:**
- React 19 + Vite 8 + **JavaScript (ES6+)** — Không dùng TypeScript
- TanStack Query (React Query) — Server state
- Zustand — Global App state (Auth)
- React Hook Form + Zod — Form & validation
- Axios — HTTP Client
- TailwindCSS + ShadcnUI — Styling

**Context Files cần nạp vào AI:**
- `coding-conventions.md` — Quy tắc tổng thể (Naming, Structure, Frontend Standards)
- `api-design-rules.md` — Quy tắc API response (`ApiResponse<T>`, `PagedResult<T>`)
- `database-migration-conventions.md` — Mapping entity ↔ types

---

### P_FE2: Axios API Client (Core HTTP Layer)
**Context Files:** `coding-conventions.md` (Section 6.3), `api-design-rules.md`

> **Prompt:**
> "Nhiệm vụ: Setup cầu nối API cốt lõi cho CLS Frontend.
>
> Tạo file `src/shared/services/apiClient.js`:
> 1. Khởi tạo Axios instance với `baseURL` đọc từ `import.meta.env.VITE_API_BASE_URL` (mặc định: `http://localhost:5000/api/v1`).
> 2. **Request Interceptor:** Tự động đọc `accessToken` từ Zustand auth store và đính vào `Authorization: Bearer <token>` Header. Nếu không có token thì bỏ qua.
> 3. **Response Interceptor (Unwrap ApiResponse):**
>    - Định dạng response từ Backend luôn là `{ success: bool, message: string, data: T }`.
>    - Interceptor phải tự động unwrap: chỉ trả về `response.data.data` cho các lệnh gọi service.
>    - Nếu `success === false`, ném ra `Error(response.data.message)`.
>    - Nếu HTTP status là **401**: dispatch action logout khỏi Zustand store và redirect về `/login`.
>    - Nếu HTTP status là **403**: Ném ra lỗi 'Bạn không có quyền thực hiện thao tác này.'
> 4. Export dưới dạng singleton object: `export const apiClient = axiosInstance;`
>
> Lưu ý: Service functions KHÔNG được dùng try/catch — lỗi phải được bubble up để hook/component xử lý."

### P_FE3: Global Stores & Types (Auth + Shared Constants)
**Context Files:** `coding-conventions.md` (Section 6.4), `api-design-rules.md`

> **Prompt:**
> "Nhiệm vụ: Xây dựng Global State và các hằng số dùng chung cho toàn dự án CLS.
>
> 1. Tạo Zustand store `src/app/provider/authStore.js`:
>    - State: `{ user: null, accessToken: null, refreshToken: null, isAuthenticated: false }`
>    - Actions: `login(userData, tokens)`, `logout()`, `updateToken(newAccessToken)`
>    - Persist state vào `localStorage` bằng middleware `persist` của Zustand (chỉ persist `accessToken` và `refreshToken`, không persist `user` object toàn bộ vì lý do bảo mật).
>
> 2. Tạo file `src/shared/utils/constants.js` chứa:
>    - `USER_ROLES`: `{ ADMIN: 'Admin', TEACHER: 'Teacher' }`
>    - `ROUTE_PATHS`: `{ LOGIN: '/login', DASHBOARD: '/', STUDENTS: '/students', CLASSES: '/classes', SESSIONS: '/sessions', ATTENDANCE: '/attendance' }`
>    - `STATUS`: `{ ACTIVE: 'active', INACTIVE: 'inactive' }`
>
> 3. Tạo file `src/shared/utils/formatters.js` chứa các utility functions:
>    - `formatDate(isoString)`: Format ISO string → `DD/MM/YYYY`
>    - `formatDateTime(isoString)`: Format → `DD/MM/YYYY HH:mm`
>    - `formatCurrency(amount)`: Format → `1.500.000 ₫` (Việt Nam locale)"

### P_FE4: Routing, Layouts & Guard System
**Context Files:** `coding-conventions.md` (Section 6.5), Zustand auth store từ bước trước

> **Prompt:**
> "Nhiệm vụ: Xây dựng App Shell, hệ thống điều hướng và bảo vệ route cho CLS.
>
> 1. Tạo `src/app/guards/PrivateRoute.jsx`:
>    - Đọc `isAuthenticated` từ Zustand auth store.
>    - Nếu chưa đăng nhập: redirect về `/login` (dùng `<Navigate to='/login' replace />`).
>    - Nếu có `requiredRole` prop: kiểm tra thêm role của user, không đúng thì redirect về `/` với thông báo lỗi.
>
> 2. Tạo `src/app/layouts/MainLayout.jsx`:
>    - Cấu trúc: `Topbar (header)` + `Sidebar` bên trái + `<Outlet />` (nội dung chính).
>    - Topbar hiển thị: tên ứng dụng "CLS" + Avatar của user + nút Logout.
>    - Sidebar hiển thị menu: Dashboard, Học sinh, Lớp học, Lịch học, Điểm danh.
>    - Dùng TailwindCSS, layout responsive: Sidebar ẩn đi trên mobile, hiển thị khi ≥ `lg`.
>
> 3. Tạo `src/app/layouts/AuthLayout.jsx`: Layout đơn giản căn giữa, dùng cho trang Login.
>
> 4. Tạo `src/app/routers/AppRouter.jsx` với toàn bộ routes:
>    ```
>    /login            → LoginPage (AuthLayout, public)
>    /                 → DashboardPage (MainLayout, PrivateRoute)
>    /students         → StudentsPage (PrivateRoute)
>    /students/:id     → StudentDetailPage (PrivateRoute)
>    /classes          → ClassesPage (PrivateRoute)
>    /sessions         → SessionsPage (PrivateRoute)
>    /attendance       → AttendancePage (PrivateRoute)
>    ```
>
> 5. Kết nối vào `App.jsx` và wrap toàn bộ app với `QueryClientProvider` (TanStack Query) và `BrowserRouter`."

### P_FE5: Feature Service & Custom Hook Layer (Data Access)
**Context Files:** `api-design-rules.md`, Swagger/endpoint của feature tương ứng từ Backend

> **Prompt:**
> "Nhiệm vụ: Code layer kết nối data cho Feature: **[TÊN_FEATURE]** (Ví dụ: student, class, session, attendance).
>
> 1. Tạo file service `src/features/[feature]/services/[feature]Service.js`:
>    - Gọi toàn bộ endpoints của feature qua `apiClient` (đã unwrap sẵn).
>    - Bao gồm CRUD và các endpoint nghiệp vụ riêng (Ví dụ: `studentService.getPackages(id)`).
>    - Theo chuẩn `api-design-rules.md`: tất cả params phân trang là `{ page, pageSize }`.
>
> 2. Tạo file custom hook `src/features/[feature]/hooks/use[Feature].js` bằng TanStack Query:
>    - `useQuery` cho các lệnh GET (list & detail), với `queryKey` chuẩn là `['[feature]', params]`.
>    - `useMutation` cho POST, PUT, DELETE — sau khi thành công phải gọi `queryClient.invalidateQueries(['[feature]'])` để refresh cache.
>    - Return đủ: `{ data, isLoading, error, createFn, updateFn, deleteFn }`.
>
> 3. Tạo Zod schema `src/features/[feature]/schemas/[feature].schema.js`:
>    - Define `create[Feature]Schema` và `update[Feature]Schema` khớp với request DTO của Backend.
>    - Validation messages phải bằng Tiếng Việt."

### P_FE7: Frontend AI Code Review
**Context Files:** Code của Feature vừa làm, `coding-conventions.md` (Section 6, 9)

> **Prompt:**
> "Đóng vai Senior Frontend Reviewer. Hãy dò quét TẤT CẢ các file React vừa code cho Feature: **[TÊN_FEATURE]**.
>
> Checklist bắt buộc:
> 1. **Prop Drilling:** Có truyền props sâu quá 2 cấp không? Nếu có, đề xuất dùng Zustand hoặc React Context.
> 2. **Axios trong UI Component:** Có gọi `apiClient`/`axios` trực tiếp bên trong `useEffect` của component không? Vi phạm nghiêm trọng — phải chuyển về custom hook.
> 3. **TanStack Query:** Có dùng `useEffect + useState` để fetch data không? Phải thay bằng `useQuery`.
> 4. **Form:** Có dùng `onChange` thủ công thay vì React Hook Form không?
> 5. **Key prop:** Có dùng `index` làm `key` khi render list không? Phải dùng entity `id`.
> 6. **Missing States:** Có component nào thiếu xử lý `isLoading` hoặc `error` state không?
>
> Hãy tự động generate đoạn mã refactor để sửa chữa cho tất cả vi phạm phát hiện được."
