# Full Implementation Workflow Prompts — CLS Project

> **Tech Stack:** ASP.NET Core 10 · EF Core 10 · PostgreSQL (Supabase) · FluentValidation · AutoMapper · Serilog · xUnit + Moq  
> **Architecture:** Layered Monolith — `CLS.Server` / `CLS.BLL` / `CLS.DAL`  
> **Adapted from:** `08_promt_full_implementation_workflow.md` (ATS/Java version)

## PHASE 1: INFRASTRUCTURE-FIRST (ONE-TIME SETUP)

_Thực hiện các Prompt P1 đến P6 tuần tự để xây dựng nền móng (Foundation Layer). Quá trình này **CHỈ THỰC HIỆN 1 LẦN** lúc bắt đầu dự án. Bối cảnh cần cung cấp thay đổi theo từng bài tập._

### P1: Project Scaffolding

**Context Files Attached:** `project_context.md` (L1), `.ai-context/coding-conventions-backend.md` (L3), `Documents/05_ADR/ADR-001-adopt-modular-monolith-dotnet-react.md`

> **Prompt:**
> "Bám sát L1, L3 và bản thiết kế kiến trúc ADR-001. Với vai trò Senior .NET Architect, hãy tạo toàn bộ khung project scaffolding cho dự án **CLS** (Classroom Management System).
> 1. Tạo Solution `.sln` và 3 project con theo đúng chuẩn Layered Architecture:
>    - `CLS.Server` — ASP.NET Core 10 Web API

>    - `CLS.DAL` — Class Library (Data Access Layer)
> 2. Tạo toàn bộ cấu trúc thư mục theo coding conventions L3 (tạo thư mục rỗng với `.gitkeep`).
> 3. Tạo `appsettings.json` + `appsettings.Development.json`, `Program.cs` (pipeline DI đầy đủ), cấu hình **Swagger/OpenAPI** chỉ bật trong môi trường Development.
> 4. Cấu hình NuGet packages: `Microsoft.EntityFrameworkCore`, `Npgsql.EntityFrameworkCore.PostgreSQL`, `AutoMapper`, `FluentValidation.AspNetCore`, `Serilog.AspNetCore`, `Microsoft.AspNetCore.Authentication.JwtBearer`.
>
> Không tạo Entity hay Logic cụ thể nào. Trả về cấu trúc cây thư mục và nội dung các file gốc."

### P2: BaseEntity

**Context Files Attached:** `.ai-context/coding-conventions-backend.md` (L3)

> **Prompt:**
> "Dựa vào quy chuẩn Coding L3 (mục 10 — Database & EF Core Conventions), hãy tạo `BaseEntity.cs` trong `CLS.DAL/Entities/`.
>
> Yêu cầu bắt buộc:
> - Primary key kiểu `int` (Identity) hoặc `Guid` — chọn theo ERD của dự án.
> - Các trường audit: `CreatedAt`, `UpdatedAt` (tự động cập nhật qua EF Core Interceptor hoặc `SaveChanges` override).
> - Soft delete: `IsDeleted` (`bool`, default `false`) + `DeletedAt` (`DateTime?`).
> - Tuyệt đối **KHÔNG** dùng Data Annotations trên Entity — cấu hình toàn bộ qua Fluent API trong `IEntityTypeConfiguration<T>`.
> - Áp dụng `HasQueryFilter(x => !x.IsDeleted)` global soft-delete filter tại `AppDbContext.OnModelCreating`."

### P3: Universal API Wrappers

**Context Files Attached:** `.ai-context/api-design-rules.md` (L2), `.ai-context/coding-conventions-backend.md` (L3)

> **Prompt:**
> "Quán triệt quy ước API từ tài liệu L2 (mục 3 & 9), hãy generate 2 class trong `CLS.BLL/Common/`:
>
> 1. `ApiResponse<T>` — generic response wrapper với fields `Code` (int), `Message` (string), `Data` (T?).
>    Implement static factory methods: `Success(T data, string message, int code = 200)` và `Fail(string message, int code = 400)`.
>    Định dạng bắt buộc: `{ \"code\": 200, \"message\": \"...\", \"data\": {...} }`.
>
> 2. `PagedResult<T>` — pagination wrapper với fields: `Items` (List<T>), `TotalCount`, `Page`, `PageSize`, `TotalPages`.
>
> Đảm bảo JSON serialization tuân thủ **camelCase** (cấu hình `JsonNamingPolicy.CamelCase` trong `Program.cs`)."

### P4: Exception Hierarchy & Global Middleware

**Context Files Attached:** `.ai-context/api-design-rules.md` (L2), `.ai-context/coding-conventions-backend.md` (L3), `ApiResponse.cs` (đã có)

> **Prompt:**
> "Triển khai hệ thống xử lý exception cho CLS:
>
> 1. Tạo các Custom Exception class trong `CLS.BLL/Common/Exceptions/`:
>    - `NotFoundException` → HTTP 404
>    - `ValidationException` → HTTP 400
>    - `ConflictException` → HTTP 409 (dùng cho scheduling conflict)
>    - `UnauthorizedException` → HTTP 401
>    - `ForbiddenException` → HTTP 403
>
> 2. Tạo `ExceptionHandlingMiddleware.cs` trong `CLS.Server/Middlewares/` sử dụng `IMiddleware`.
>    Middleware bắt tất cả unhandled exception và map về `ApiResponse<object>.Fail(...)` thống nhất.
>    **Không bao giờ** leak stack trace về client trong môi trường Production.
>
> 3. Đăng ký middleware trong `Program.cs` trước mọi middleware khác."

### P5: Security Configuration (JWT)

**Context Files Attached:** `.ai-context/api-design-rules.md` (L2), `.ai-context/coding-conventions-backend.md` (L3, mục 11)

> **Prompt:**
> "Hãy setup JWT Authentication & Authorization cho CLS:
>
> 1. Cấu hình `AddAuthentication().AddJwtBearer(...)` trong `Program.cs` với:
>    - Algorithm: HS256
>    - Issuer & Audience đọc từ `appsettings.json` → `JwtSettings:Issuer`, `JwtSettings:Audience`
>    - Access Token expiry: 60 phút
>    - Refresh Token expiry: 7 ngày
>
> 2. Tạo `JwtService.cs` trong `CLS.BLL/Services/` với method:
>    - `GenerateAccessToken(User user)` → string
>    - `GenerateRefreshToken()` → string
>    - `ValidateToken(string token)` → ClaimsPrincipal?
>
> 3. Định nghĩa Role constants trong `CLS.Server/` hoặc `CLS.BLL/Common/`:
>    - `\"Admin\"`, `\"Teacher\"`, `\"Parent\"`
>
> Đọc danh sách public endpoints từ L2 (mục 4.3) để chia `[AllowAnonymous]` vs `[Authorize]`.
> Chưa implement User lookup — chỉ tạo skeleton đủ để filter token."

### P6: Database Base Migrations (EF Core)

**Context Files Attached:** `.ai-context/coding-conventions-backend.md` (L3, mục 10.3), Logical ERD

> **Prompt:**
> "Với tư cách Database Architect, hãy thiết lập nền tảng database cho CLS bằng EF Core Migrations:
>
> 1. Tạo `AppDbContext.cs` trong `CLS.DAL/Data/` với:
>    - Kết nối Npgsql (PostgreSQL/Supabase)
>    - Override `OnModelCreating` để apply tất cả `IEntityTypeConfiguration<T>` từ assembly
>    - Override `SaveChangesAsync` để tự động set `CreatedAt`, `UpdatedAt`, `IsDeleted`
>
> 2. Tạo Migration đầu tiên cho bảng nền tảng (dựa theo ERD):
>    - `InitialCreate` — tạo bảng `users` (cốt lõi: Id, FullName, Email, PasswordHash, Role, audit fields, IsDeleted)
>
> 3. Viết lệnh CLI chuẩn để chạy migration:
>    ```bash
>    dotnet ef migrations add InitialCreate --project CLS.DAL --startup-project CLS.Server
>    dotnet ef database update --project CLS.DAL --startup-project CLS.Server
>    ```
>
> Tuân thủ: Table & column names dùng `snake_case` (PostgreSQL convention), cấu hình qua Fluent API."

## 🚀 PHASE 2: VERTICAL SLICE IMPLEMENTATION

_Thực hiện lặp lại vòng lặp **P7 → P12** cho **từng tính năng / User Story** riêng biệt (ví dụ: Quản lý Student Enrollment, Đặt lịch Class Session, Submit Attendance, Renewal Alert). "Vertical slice" nạp Context tập trung hơn và phân làm 4 chặng: Data → Business → API → QA._

---

### P7: Data Layer (Entity & Migration)

**Context Files Attached:** `.ai-context/coding-conventions-backend.md` (L3), Logical ERD

> **Prompt:**
> "Nhiệm vụ: Hiện thực hóa Vertical Slice cho tính năng: **[TÊN_FEATURE]**.
>
> Dựa vào Database Conventions (L3, mục 10) và ERD Logical, hãy generate:
>
> 1. **Entity class** trong `CLS.DAL/Entities/`:
>    - Extend từ `BaseEntity.cs`
>    - Khai báo navigation properties cho tất cả quan hệ với `= null!` (non-nullable reference)
>
> 2. **Fluent API Configuration** class `[Entity]Configuration.cs` trong `CLS.DAL/Configurations/`:
>    - `ToTable(\"snake_case_name\")`
>    - Cấu hình tất cả FK với `OnDelete(DeleteBehavior.Restrict)` (explicit)
>    - Áp dụng `HasQueryFilter(x => !x.IsDeleted)` nếu entity có soft delete
>
> 3. **EF Core Migration** với tên có nghĩa (e.g., `AddClassSessionTable`):
>    ```bash
>    dotnet ef migrations add Add[Feature]Table --project CLS.DAL --startup-project CLS.Server
>    ```"

### P8: Persistence & Transfer Layer (Repository / DTOs)

**Context Files Attached:** API Specs (Swagger/OpenAPI), `.ai-context/coding-conventions-backend.md` (L3)

> **Prompt:**
> "Nhiệm vụ: Cung cấp layer giao tiếp cho tính năng **[TÊN_FEATURE]**.
>
> 1. **Repository** trong `CLS.DAL/Repositories/`:
>    - Interface `I[Feature]Repository` extends `IRepository<TEntity>` (generic base)
>    - Implementation `[Feature]Repository` với các method custom (e.g., `FindByEmailAsync`, `GetNearExpiryAsync`)
>    - Dùng `AsNoTracking()` cho mọi read-only query
>    - Đăng ký DI trong `Program.cs`: `builder.Services.AddScoped<I[Feature]Repository, [Feature]Repository>()`
>
> 2. **DTOs** trong `CLS.BLL/DTOs/`:
>    - `Create[Feature]Request` — request DTO có FluentValidation annotations tương ứng
>    - `Update[Feature]Request` — partial update DTO
>    - `[Feature]Dto` / `[Feature]Response` — response DTO (KHÔNG expose sensitive fields như PasswordHash)
>
> 3. **AutoMapper Profile** `[Feature]MappingProfile.cs` trong `CLS.BLL/`:
>    - Map Entity ↔ DTO, bao gồm `ReverseMap()` khi cần"

### P9: Business Logic Layer (Service)

**Context Files Attached:** User Story / Acceptance Criteria (AC), Sequence Diagrams, `.ai-context/coding-conventions-backend.md` (L3)

> **Prompt:**
> "Nhiệm vụ: Xây dựng bộ não nghiệp vụ cho tính năng **[TÊN_FEATURE]**.
>
> Dựa trên User Story + AC và Sequence Diagram (nếu có), hãy cài đặt:
>
> 1. **Interface** `I[Feature]Service.cs` trong `CLS.BLL/Interfaces/`
>
> 2. **Service Implementation** `[Feature]Service.cs` trong `CLS.BLL/Services/`:
>    - Constructor injection: `I[Feature]Repository`, `IMapper`, `ILogger<[Feature]Service>` (Serilog)
>    - Implement đầy đủ luồng Happy Path theo AC
>    - Throw Custom Exceptions khi vi phạm AC: `NotFoundException`, `ConflictException`, v.v.
>    - Dùng **structured logging**: `_logger.LogInformation(\"Action {Entity} {Id}\", ...)`
>    - **KHÔNG** gọi trực tiếp Repository của module khác — cross-module qua Service Interface
>    - Tránh 'God Classes': tách Helper/Validator nếu method > 30 dòng
>
> 3. **FluentValidation Validator** `Create[Feature]RequestValidator.cs` trong `CLS.BLL/Validators/`"

### P10: API Surface (Controller)

**Context Files Attached:** API Specs, `.ai-context/api-design-rules.md` (L2), `.ai-context/coding-conventions-backend.md` (L3)

> **Prompt:**
> "Nhiệm vụ: Publish API Endpoint cho tính năng **[TÊN_FEATURE]**.
>
> Căn cứ API rules (L2) và API specs, tạo `[Feature]Controller.cs` trong `CLS.Server/Controllers/`:
>
> - Attribute: `[ApiController]`, `[Route(\"api/v1/[controller]\")]`
> - Authorization: `[Authorize(Roles = \"...\")]` đúng theo API specs; `[AllowAnonymous]` chỉ cho auth endpoints
> - URL: kebab-case, plural nouns (e.g., `/api/v1/class-sessions`, `/api/v1/students`)
> - Mọi response phải wrap qua `ApiResponse<T>.Success(...)` hoặc `ApiResponse<object>.Fail(...)`
> - Mọi action phải có `[ProducesResponseType]` + XML doc comment cho Swagger
> - Controller chỉ ủy quyền sang `I[Feature]Service` — **KHÔNG** nhúng business logic hay gọi DbContext trực tiếp
>
> Ví dụ pattern chuẩn:
> ```csharp
> [HttpGet(\"{id}\")]
> [ProducesResponseType(typeof(ApiResponse<StudentDto>), 200)]
> [ProducesResponseType(typeof(ApiResponse<object>), 404)]
> public async Task<IActionResult> GetById(int id)
> {
>     var result = await _studentService.GetByIdAsync(id);
>     return Ok(ApiResponse<StudentDto>.Success(result, \"Student retrieved successfully\"));
> }
> ```"

## 🔍 PHASE 3: QA AND DELIVERY

_Đây là bước AI Augment Review (AI đóng vai Tester và Code Reviewer) nhằm đảm bảo sự minh bạch của Vertical Slice trước khi commit._

### P11: AI Code Review (Clean Code & SOLID)

**Context Files Attached:** Toàn bộ codebase của Slice vừa lập trình, `.ai-context/coding-conventions-backend.md` (L3 — mục 13 Anti-patterns)

> **Prompt:**
> "Đóng vai là Tech Lead .NET, hãy kiểm tra chéo (review) toàn bộ Vertical Slice **[TÊN_FEATURE]** vừa cung cấp.
>
> Kiểm tra Checklist sau:
> - **N+1 Query problem**: Các navigation property có được Include đúng chỗ không? `AsNoTracking()` có được dùng cho read-only query không?
> - **Data leakage**: Các field nhạy cảm (`PasswordHash`, token, PII) có rơi vào DTO response không?
> - **Entity integrity**: Entity có extend đúng `BaseEntity`? Fluent API config có đủ `HasQueryFilter` soft-delete không?
> - **Async compliance**: Mọi I/O operation có dùng `async/await` đúng không? Có `.Result` hay `.Wait()` blocking không?
> - **Logging**: Có dùng string interpolation trong log không? (phải dùng structured message template)
> - **SOLID violations**: Service có quá nhiều trách nhiệm không? Interface có bị vi phạm ISP không?
>
> Trả về danh sách issue nếu có và đề xuất code refactor cụ thể."

### P12: Unit Testing Generation (BDD / xUnit)

**Context Files Attached:** User Story / AC, Slice Codebase, `.ai-context/coding-conventions-backend.md` (L3, mục 14)

> **Prompt:**
> "Ánh xạ trực tiếp Acceptance Criteria dạng **Given-When-Then** thành cấu trúc **xUnit + Moq** tests cho Service `[Feature]Service` vừa làm.
>
> Yêu cầu:
> - Test project: `CLS.Tests` (đặt trong solution)
> - Naming pattern: `MethodName_Scenario_ExpectedResult`
>   - Ví dụ: `GetByIdAsync_StudentNotFound_ThrowsNotFoundException`
>   - Ví dụ: `CreateAsync_DuplicateEmail_ThrowsConflictException`
> - Coverage tối thiểu **70% cho Service**, **100% cho Validator**
> - Bắt buộc viết test cho **Happy Path** AND **Exception branches** (NotFound, Conflict, Validation failures)
> - Mock `IRepository` bằng Moq, **KHÔNG** mock EF Core DbContext trực tiếp
>
> Template cơ bản:
> ```csharp
> public class StudentServiceTests
> {
>     private readonly Mock<IStudentRepository> _repoMock = new();
>     private readonly Mock<IMapper> _mapperMock = new();
>     private readonly StudentService _sut;
>
>     public StudentServiceTests()
>     {
>         _sut = new StudentService(_repoMock.Object, _mapperMock.Object, ...);
>     }
>
>     [Fact]
>     public async Task GetByIdAsync_StudentNotFound_ThrowsNotFoundException()
>     {
>         // Arrange
>         _repoMock.Setup(r => r.GetByIdAsync(It.IsAny<int>())).ReturnsAsync((Student?)null);
>
>         // Act & Assert
>         await Assert.ThrowsAsync<NotFoundException>(() => _sut.GetByIdAsync(99));
>     }
> }
> ```"
