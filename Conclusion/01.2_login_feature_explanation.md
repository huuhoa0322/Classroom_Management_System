## 2. Chi tiết từng class/hàm

### 🖥️ FRONTEND

---

### `LoginPage.jsx`
**Vai trò:** Component UI — render form và handle user interaction.

```jsx
// State quản lý input
const [email, setEmail] = useState('');
const [password, setPassword] = useState('');

// Lấy mutation từ hook
const { mutate: login, isPending, error } = useLogin();

// Khi submit form
const handleSubmit = (e) => {
  e.preventDefault();
  login({ email, password });   // ← Gọi useLogin.mutate()
};
```

**Tương tác:**
- Gọi `useLogin()` hook để lấy hàm `login` và trạng thái (`isPending`, `error`)
- Khi submit → gọi `login({ email, password })`
- Hiển thị spinner khi `isPending = true`
- Hiển thị error banner khi `error` có giá trị

---

### `useLogin` (hook trong `useAuth.js`)
**Vai trò:** Business logic layer của FE — kết nối UI với API call và store.

```js
export const useLogin = () => {
  const navigate   = useNavigate();
  const storeLogin = useAuthStore((s) => s.login); // Lấy action từ Zustand

  return useMutation({
    // 1. Gọi API
    mutationFn: (credentials) => authService.login(credentials),

    // 2. Xử lý khi thành công
    onSuccess: (data) => {
      // data = LoginResponse đã được unwrap bởi apiClient
      storeLogin(data.user, {
        accessToken:  data.accessToken,
        refreshToken: data.refreshToken,
      });
      navigate('/', { replace: true });
    },
  });
};
```

**Tương tác:**
- Nhận `credentials` từ `LoginPage` qua `mutate()`
- Gọi `authService.login()` thông qua `mutationFn`
- Khi thành công: gọi `storeLogin()` của Zustand, rồi redirect
- TanStack Query tự động quản lý loading/error state → cập nhật ngược lại `LoginPage`

---

### `authService.js`
**Vai trò:** Data access layer của FE — định nghĩa các API call liên quan đến auth.

```js
export const authService = {
  login: (credentials) => apiClient.post('/auth/login', credentials),
  //                       └── Trả về Promise<LoginResponse>
};
```

**Tương tác:**
- Nhận `credentials = { email, password }` từ `useLogin`
- Gọi `apiClient.post()` — Axios instance đã được cấu hình sẵn
- Trả về Promise chứa kết quả đã unwrap từ `ApiResponse<LoginResponse>`

---

### `apiClient.js` (Axios instance)
**Vai trò:** HTTP client trung tâm — xử lý auth header, unwrap response, handle lỗi toàn cục.

#### Request Interceptor
```js
axiosInstance.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`; // Đính token vào HEADER
  }
  return config;
});
```
> Chạy TRƯỚC mỗi request. Đọc token từ Zustand store và gắn vào header.
> Khi login: token = null → không có Authorization header (đúng, vì chưa đăng nhập).
> Các request sau khi login: token có giá trị → header được đính tự động.

#### Response Interceptor
```js
axiosInstance.interceptors.response.use((response) => {
  const apiResponse = response.data; // { code, message, data }

  if ('code' in apiResponse) {
    if (apiResponse.code < 200 || apiResponse.code >= 300)
      return Promise.reject(new Error(apiResponse.message));

    return apiResponse.data; // ← UNWRAP: Trả về chỉ phần data
  }
  return response.data;
});
```
> Chạy SAU mỗi response. Bóc tách `ApiResponse<T>` → chỉ trả về phần `data`.
> `useLogin.onSuccess(data)` nhận được `LoginResponse` trực tiếp, không phải `ApiResponse`.

### `authStore.js` (Zustand Store)
**Vai trò:** Global state management — lưu trữ và persist thông tin auth.

```js
export const useAuthStore = create(
  persist(
    (set) => ({
      // State
      user: null,
      accessToken: null,
      isAuthenticated: false,
      _hasHydrated: false,

      // Actions
      login: (userData, tokens) => set({
        user: userData,
        accessToken: tokens.accessToken,
        refreshToken: tokens.refreshToken,
        isAuthenticated: true,
      }),

      logout: () => set({ user: null, accessToken: null, isAuthenticated: false }),

      setHasHydrated: (val) => set({ _hasHydrated: val }),
    }),
    {
      name: 'cls-auth-storage',        // Key trong localStorage
      partialize: (state) => ({        // Chỉ persist token, không persist user
        accessToken:     state.accessToken,
        refreshToken:    state.refreshToken,
        isAuthenticated: state.isAuthenticated,
      }),
      onRehydrateStorage: () => (state) => {
        state?.setHasHydrated(true);   // Đánh dấu đã đọc xong localStorage
      },
    }
  )
);
```

**`_hasHydrated` — tại sao cần:**
```
Page reload → Zustand khởi tạo với { accessToken: null } (default)
            → Bắt đầu đọc localStorage (async)
            → Trong lúc chờ: _hasHydrated = false
            → PrivateRoute hiển thị spinner (không redirect sai)
            → Đọc xong: _hasHydrated = true, accessToken = "eyJ..."
            → PrivateRoute render đúng route được bảo vệ
```

### 🔧 BACKEND

### `AuthController`
**Vai trò:** HTTP entry point — nhận request, gọi service, trả response chuẩn.

```csharp
[ApiController]
[Route("api/v1/auth")]
public class AuthController : ControllerBase
{
    private readonly IAuthService _authService;

    [HttpPost("login")]
    [AllowAnonymous]   // Endpoint công khai, không cần JWT
    public async Task<IActionResult> Login(
        [FromBody] LoginRequest request,
        CancellationToken ct = default)
    {
        var result = await _authService.LoginAsync(request, ct);
        return Ok(ApiResponse<LoginResponse>.Success(result, "Đăng nhập thành công."));
    }
}
```

**Tương tác:**
- Inject `IAuthService` qua constructor (DI)
- Deserialize request body → `LoginRequest { Email, Password }`
- Gọi `_authService.LoginAsync()` → nhận `LoginResponse`
- Wrap kết quả vào `ApiResponse<T>` → trả về HTTP 200

---

### `AuthService` (implements `IAuthService`)
**Vai trò:** Business logic — kiểm tra credentials và sinh tokens.

```csharp
public async Task<LoginResponse> LoginAsync(LoginRequest request, CancellationToken ct)
{
    // BƯỚC 1: Tìm user theo email (case-insensitive, chỉ active)
    var user = await _ctx.Users
        .AsNoTracking()
        .FirstOrDefaultAsync(
            u => u.Email.ToLower() == request.Email.ToLower()
              && u.Status == "active", ct);

    // BƯỚC 2: Kiểm tra user tồn tại
    if (user is null)
        throw new UnauthorizedException("Email hoặc mật khẩu không chính xác.");
    //  └── ExceptionHandlingMiddleware bắt → trả HTTP 401 JSON

    // BƯỚC 3: Verify password với BCrypt
    var isValid = BCrypt.Net.BCrypt.Verify(request.Password, user.PasswordHash);
    if (!isValid)
        throw new UnauthorizedException("Email hoặc mật khẩu không chính xác.");
    //  └── Cùng message để tránh user enumeration attack

    // BƯỚC 4: Sinh tokens
    var jwtUser = new JwtUserDto(user.Id, user.Email, user.Role, user.FullName);
    var accessToken  = _jwtService.GenerateAccessToken(jwtUser);
    var refreshToken = _jwtService.GenerateRefreshToken();

    // BƯỚC 5: Trả về kết quả
    return new LoginResponse
    {
        AccessToken  = accessToken,
        RefreshToken = refreshToken,
        User = new UserInfo { Id = user.Id, FullName = user.FullName,
                              Email = user.Email, Role = user.Role }
    };
}
```

**Tương tác:**
- `AppDbContext` (EF Core) → truy vấn bảng `users`
- `BCrypt.Net.BCrypt.Verify()` → so khớp password plain-text với hash trong DB
- `IJwtService.GenerateAccessToken()` → tạo JWT
- `IJwtService.GenerateRefreshToken()` → tạo refresh token ngẫu nhiên

---

### `JwtService` (implements `IJwtService`)
**Vai trò:** Tạo và validate JWT token.

```csharp
// Tạo Access Token (có thời hạn 60 phút)
public string GenerateAccessToken(JwtUserDto user)
{
    var descriptor = new SecurityTokenDescriptor
    {
        Subject = new ClaimsIdentity([
            new Claim(JwtRegisteredClaimNames.Sub,   user.Id.ToString()),
            new Claim(JwtRegisteredClaimNames.Email, user.Email),
            new Claim(JwtRegisteredClaimNames.Name,  user.FullName),
            new Claim(ClaimTypes.Role,               user.Role),  // ← Dùng cho [Authorize(Roles)]
            new Claim(JwtRegisteredClaimNames.Jti,   Guid.NewGuid().ToString()),
        ]),
        Issuer             = _config["JwtSettings:Issuer"],
        Audience           = _config["JwtSettings:Audience"],
        Expires            = DateTime.UtcNow.AddMinutes(60),
        SigningCredentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256)
    };
    return _tokenHandler.CreateToken(descriptor); // JsonWebTokenHandler (.NET 10)
}

// Tạo Refresh Token (64 bytes ngẫu nhiên)
public string GenerateRefreshToken()
{
    return Convert.ToBase64String(RandomNumberGenerator.GetBytes(64));
}
```

**JWT Claims được đưa vào token:**
| Claim | Giá trị | Mục đích |
|-------|---------|---------|
| `sub` | user.Id | Định danh user |
| `email` | user.Email | Thông tin hiển thị |
| `name` | user.FullName | Thông tin hiển thị |
| `role` | "Admin" / "Teacher" | Kiểm tra `[Authorize(Roles)]` |
| `jti` | GUID ngẫu nhiên | Token ID (chống replay) |

## 4. Design Decisions quan trọng

### BCrypt — tại sao không dùng SHA256/MD5?
BCrypt tích hợp **salt ngẫu nhiên** và **cost factor** (số vòng hash). Cùng password nhưng mỗi lần hash ra một chuỗi khác nhau → không thể dùng **rainbow table** để crack.

### Cùng error message cho sai email và sai password
```csharp
// Sai email:
throw new UnauthorizedException("Email hoặc mật khẩu không chính xác.");
// Sai password:
throw new UnauthorizedException("Email hoặc mật khẩu không chính xác.");
```
Nếu báo riêng "Email không tồn tại", hacker biết email đó có trong hệ thống → **user enumeration attack**. Cùng message → không lộ thông tin.

### Tại sao dùng `IAuthService` interface thay vì `AuthService` trực tiếp?
**Dependency Inversion** — Controller phụ thuộc vào abstraction, không phụ thuộc vào implementation cụ thể. Dễ thay thế khi viết Unit Test (mock `IAuthService`).

### `partialize` trong Zustand persist — tại sao không persist `user`?
`user` object có thể chứa thông tin nhạy cảm và có thể outdated (vai trò thay đổi, v.v.). Token đã đủ để xác thực; `user` được lấy từ JWT claims hoặc gọi `/me` endpoint nếu cần.

### `AsNoTracking()` trong query user
```csharp
var user = await _ctx.Users.AsNoTracking()...
```
Chỉ đọc, không cần EF Core track changes → tiết kiệm memory và tăng performance cho read-only queries.
