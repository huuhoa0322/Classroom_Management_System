## 2. Các file đã chỉnh sửa

### Backend

#### `CLS.Server/Program.cs`
- **Đăng ký `IAuthService`** trong DI container
- **Thêm CORS** `SetIsOriginAllowed(_ => true)` cho môi trường Development (chấp nhận mọi Vite port)
- **Tắt HTTPS redirect trong dev** (`UseHttpsRedirection` chỉ chạy ở Production) — fix lỗi 307 redirect phá POST request
- **Thêm `RoleClaimType` + `NameClaimType`** vào `TokenValidationParameters` để JWT role authorization hoạt động đúng với `JsonWebTokenHandler` (.NET 10)
- **Xóa đăng ký `AddAutoMapper` trùng lặp**

#### `CLS.Server/Controllers/StudentsController.cs`
- Giữ nguyên `[Authorize(Roles = "Admin")]` — xác nhận hoạt động đúng sau khi fix auth flow

#### `Documents/02_CLS_Seed_Data.sql`
- Cập nhật hash BCrypt từ placeholder giả (`$2a$11$eEON...`) sang hash thật (`$2b$11$EL.r...`) cho password `Password123!`

### Frontend

#### `cls.client/.env.local`
```
VITE_API_BASE_URL=/api/v1
```
Dùng đường dẫn tương đối để request đi qua Vite proxy (không gọi thẳng backend).

#### `cls.client/vite.config.js`
Thêm proxy rule cho `/api`:
```js
'^/api': {
    target,        // https://localhost:7065
    secure: false,
    changeOrigin: true
}
```
→ Mọi request `/api/*` từ browser được Vite forward sang backend, bỏ qua SSL cert check.

#### `cls.client/src/shared/services/apiClient.js` ⭐ Fix cốt lõi
```js
// TRƯỚC (sai): kiểm tra 'success' field không tồn tại trong ApiResponse<T>
if ('success' in apiResponse) { ... }

// SAU (đúng): kiểm tra 'code' field — khớp với ApiResponse<T> thực tế của backend
if ('code' in apiResponse) {
  if (apiResponse.code < 200 || apiResponse.code >= 300) reject(...)
  return apiResponse.data   // Unwrap: LoginResponse, PagedResult, etc.
}
```

#### `cls.client/src/app/provider/authStore.js`
Thêm **Zustand hydration guard**:
```js
_hasHydrated: false,
setHasHydrated: (state) => set({ _hasHydrated: state }),

// Trong persist config:
onRehydrateStorage: () => (state) => {
  state?.setHasHydrated(true);
}
```

#### `cls.client/src/app/guards/PrivateRoute.jsx`
Chờ hydration hoàn tất trước khi kiểm tra `isAuthenticated`:
```jsx
if (!_hasHydrated) return <Spinner />;   // Tránh flash redirect sai
if (!isAuthenticated) return <Navigate to="/login" />;
```

#### `cls.client/src/app/layouts/AuthLayout.jsx`
Đơn giản hóa thành `<Outlet />` vì `LoginPage` tự render full-screen layout.

## 3. Các bug đã phát hiện và fix

| # | Bug | Nguyên nhân | Fix |
|---|-----|-------------|-----|
| 1 | **Token NULL sau login** — root cause | `apiClient` check `'success'` field nhưng `ApiResponse<T>` chỉ có `'code'` | Đổi điều kiện unwrap sang check `'code'` |
| 2 | **307 Redirect phá POST** | `UseHttpsRedirection` redirect http→https, POST mất body | Tắt redirect khi `IsDevelopment()` |
| 3 | **Network Error từ FE** | FE gọi HTTPS trực tiếp, SSL self-signed cert bị browser block | Dùng Vite proxy `/api` (server-to-server bypass SSL) |
| 4 | **Flash redirect về /login khi reload** | Zustand persist hydrate bất đồng bộ, `isAuthenticated` chưa restore | Thêm `_hasHydrated` guard trong `PrivateRoute` |
| 5 | **CORS block** | DevCors chỉ cho phép `localhost:5173`, không có `localhost:57264` | Đổi sang `SetIsOriginAllowed(_ => true)` cho dev |
| 6 | **BCrypt hash giả trong seed** | File SQL dùng placeholder hash không verify được | Chạy `update-passwords.mjs` sinh hash thật |
| 7 | **`AddAutoMapper` đăng ký 2 lần** | Program.cs có 2 dòng `AddAutoMapper` cho cùng assembly | Xóa dòng trùng |

## 4. Kiến trúc Auth Flow (sau khi fix)

```
[Browser] → HTTPS → [Vite Dev Server :57264]
                          │
                    Proxy /api/* → [Backend :7065]
                          │
              POST /api/v1/auth/login
              ← { code:200, message:"...", data:{ accessToken, refreshToken, user } }
                          │
              apiClient.interceptor unwrap → LoginResponse
                          │
              useLogin.onSuccess → storeLogin() → Zustand Store
              Zustand persist → localStorage { accessToken, isAuthenticated }
                          │
              navigate('/')
                          │
[GET /api/v1/students] → interceptor đọc accessToken từ Zustand
                       → Authorization: Bearer eyJ...
                       → Backend [Authorize(Roles="Admin")] ✅ → 200
```
