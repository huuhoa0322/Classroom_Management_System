## Trả lời: Swashbuckle 10.x + .NET 10 — CÓ xung đột, nhưng có thể fix

### Tại sao có xung đột?

**Swashbuckle 10.x** dùng `Microsoft.OpenApi` **2.x** — phiên bản này thay đổi kiến trúc lớn để hỗ trợ OpenAPI 3.1:

| API | Swashbuckle 7.x (đang dùng) | Swashbuckle 10.x |
|---|---|---|
| `OpenApiSecurityScheme` | Class có `Reference` property | Đã **xóa** `Reference` property |

## Production hoạt động thế nào?

### Dev vs Production

```
DEV:
Browser ──→ Vite Dev Server (:57264) ──proxy──→ ASP.NET Core (:7065)
            (Vite tồn tại, proxy hoạt động)

PRODUCTION:
Browser ──→ ASP.NET Core (một server duy nhất, ví dụ :443)
            ├── /api/* → Controllers  (MapControllers)
            └── /*     → index.html   (MapFallbackToFile)
```

### Tại sao hoạt động được?

Khi build production (`npm run build`), Vite output ra thư mục `dist/`. ASP.NET Core project đã được cấu hình serve static files từ đó:

```csharp
// Program.cs — đã có sẵn
app.UseDefaultFiles();
app.MapStaticAssets();         // Serve dist/ của React
app.MapControllers();          // Xử lý /api/*
app.MapFallbackToFile("/index.html"); // SPA routing
```

Frontend và Backend cùng chạy trên **1 domain duy nhất** → `/api/v1` là **same-origin** → không cần CORS, không cần proxy, không có SSL cert issue.

| Môi trường | `VITE_API_BASE_URL` | Cơ chế |
|------------|---------------------|--------|
| **Dev** | `/api/v1` (relative) | Vite proxy forward sang `:7065` |
| **Production** | `/api/v1` (relative, giữ nguyên) | Same-origin, ASP.NET Core tự handle |

`.env.local` đang dùng `/api/v1` (relative path) là **chuẩn** cho cả 2 môi trường — không cần đổi gì khi deploy.

> **Lưu ý duy nhất cho production:** Đảm bảo chạy `npm run build` trong CI/CD pipeline trước khi publish ASP.NET Core app, để thư mục `dist/` được copy vào output của server.
