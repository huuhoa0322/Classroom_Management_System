# CLS Coding Conventions

> **Scope:** Áp dụng cho toàn bộ codebase dự án Classroom Management System (CLS).  
> **Source:** Derived from [ADR-001](../Documents/05_ADR/ADR-001-adopt-modular-monolith-dotnet-react.md) — Modular Monolith with ASP.NET Core + React.  
> **Audience:** AI agents, fullstack developers, code reviewers.

---

## 1. Architecture Overview

CLS adopts a **3-Layer Modular Monolith** pattern:

```
[Client Layer]         React 19 + Vite 8 + JavaScript (ES6+)
      ↓ HTTPS / REST
[Presentation Layer]   ASP.NET Core 10 — Controllers, Middlewares, Filters
[Business Logic Layer] Services, DTOs, Interfaces, FluentValidation
[Data Access Layer]    EF Core 10, Repositories, Entities, Migrations
      ↓
[Database]             PostgreSQL 15+ (Supabase)
```

**Module boundary rule:** Các module KHÔNG được truy cập trực tiếp vào DbContext của module khác. Mọi cross-module access phải đi qua Service Interface.

## 4. Naming Conventions

### 4.1 Backend (.NET / C#)

| Element | Convention | Example |
|---------|-----------|---------|
| **Namespace** | PascalCase, match folder | `CLS.Server.Controllers` |
| **Class** | PascalCase | `StudentEnrollmentService` |
| **Interface** | `I` prefix + PascalCase | `IStudentEnrollmentService` |
| **Method** | PascalCase | `GetEnrollmentByIdAsync()` |
| **Async Method** | Suffix `Async` | `CreateEnrollmentAsync()` |
| **Private Field** | `_camelCase` | `_studentRepository` |
| **Property** | PascalCase | `public string FullName { get; set; }` |
| **DTO** | Suffix `Dto` or `Request`/`Response` | `EnrollmentDto`, `CreateStudentRequest` |
| **Entity** | PascalCase, singular noun | `Student`, `ClassSession` |
| **Repository** | Suffix `Repository` | `StudentRepository` |
| **Controller** | Suffix `Controller` | `StudentsController` |
| **Validator** | Suffix `Validator` | `CreateStudentRequestValidator` |
| **Migration** | Auto-generated by EF Core | `AddStudentTable`, `20260420_InitSchema` |
| **Constants** | PascalCase in static class | `AppConstants.JwtSecretKey` |

### 4.2 Frontend (React / JavaScript)

| Element | Convention | Example |
|---------|-----------|---------|
| **Component file** | PascalCase `.jsx` | `StudentTable.jsx` |
| **Hook file** | camelCase, prefix `use` | `useStudents.js` |
| **Service file** | camelCase, suffix `Service` | `studentService.js` |
| **Util file** | camelCase | `dateFormatter.js` |
| **Component name** | PascalCase | `function StudentTable()` |
| **Variable / function** | camelCase | `const fetchStudents = ...` |
| **Constant** | SCREAMING_SNAKE_CASE | `const API_BASE_URL = ...` |
| **CSS class (Tailwind)** | Use Tailwind utilities; no custom CSS unless shared token | `className="flex gap-4 p-2"` |
| **Event handler** | Prefix `handle` | `handleSubmit`, `handleDelete` |
| **Boolean prop/state** | Prefix `is`/`has`/`can` | `isLoading`, `hasError`, `canEdit` |

## 5. Backend Coding Standards

### 5.1 Controller Rules

```csharp
// ✅ CORRECT — Thin controller, delegate to service
[ApiController]
[Route("api/v1/[controller]")]
[Authorize]
public class StudentsController : ControllerBase
{
    private readonly IStudentService _studentService;

    public StudentsController(IStudentService studentService)
    {
        _studentService = studentService;
    }

    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(int id)
    {
        var result = await _studentService.GetByIdAsync(id);
        return Ok(ApiResponse.Success(result));
    }
}

// ❌ WRONG — Business logic inside controller
[HttpPost]
public async Task<IActionResult> Create(CreateStudentRequest request)
{
    var student = new Student { FullName = request.FullName }; // logic in controller
    _context.Students.Add(student);
    await _context.SaveChangesAsync();
    return Ok(student);
}
```

**Rules:**
- Controllers MUST be thin: validate → call service → return `ApiResponse`
- Controllers MUST NOT access `DbContext` directly
- All routes MUST follow `api/v1/[controller]` pattern (see `api-design-rules.md`)
- Use `[Authorize]` on protected endpoints; `[AllowAnonymous]` only on auth endpoints

### 5.2 Service Layer Rules

```csharp
// ✅ CORRECT
public class StudentService : IStudentService
{
    private readonly IStudentRepository _repo;
    private readonly IMapper _mapper;

    public StudentService(IStudentRepository repo, IMapper mapper)
    {
        _repo = repo;
        _mapper = mapper;
    }

    public async Task<StudentDto> GetByIdAsync(int id)
    {
        var entity = await _repo.GetByIdAsync(id)
            ?? throw new NotFoundException($"Student {id} not found");
        return _mapper.Map<StudentDto>(entity);
    }
}
```

**Rules:**
- Services MUST implement an interface (`IXxxService`)
- Services handle business logic, validation coordination and exception throwing
- Use `NotFoundException`, `ValidationException` custom exceptions (NOT raw `Exception`)
- Use AutoMapper for DTO ↔ Entity conversion — no manual property mapping

### 5.3 Validation Rules (FluentValidation)

```csharp
public class CreateStudentRequestValidator : AbstractValidator<CreateStudentRequest>
{
    public CreateStudentRequestValidator()
    {
        RuleFor(x => x.FullName).NotEmpty().MaximumLength(100);
        RuleFor(x => x.Email).EmailAddress().When(x => x.Email != null);
        RuleFor(x => x.PhoneNumber).Matches(@"^\d{10,11}$").WithMessage("Phone must be 10-11 digits");
    }
}
```

**Rules:**
- ALWAYS use FluentValidation — never Data Annotations for business validation
- Register validators via `AddFluentValidationAutoValidation()` in `Program.cs`
- Validators are placed in `CLS.BLL/Validators/`

### 5.4 Repository Rules

```csharp
// Generic interface
public interface IRepository<T> where T : class
{
    Task<T?> GetByIdAsync(int id);
    Task<IEnumerable<T>> GetAllAsync();
    Task AddAsync(T entity);
    Task UpdateAsync(T entity);
    Task DeleteAsync(int id);
}

// Specific extension
public interface IStudentRepository : IRepository<Student>
{
    Task<IEnumerable<Student>> GetByClassIdAsync(int classId);
    Task<IEnumerable<Student>> GetNearExpiryAsync(int daysThreshold);
}
```

**Rules:**
- Generic Repository covers CRUD; extend with specific queries in domain repos
- Repositories MUST NOT contain business logic
- `SaveChangesAsync()` is called in Services, NOT inside repositories (Unit of Work pattern)
- Use EF Core `AsNoTracking()` for read-only queries

### 5.5 Async / Await Rules

- ALL I/O-bound operations MUST be `async`/`await` — no `.Result` or `.Wait()`
- Method signatures returning Task MUST have `Async` suffix
- Use `CancellationToken` for long-running background operations

### 5.6 Exception Handling

- Global exception middleware in `Middlewares/ExceptionHandlingMiddleware.cs`
- Domain exceptions: `NotFoundException`, `ValidationException`, `ConflictException` (e.g., scheduling conflict)
- All unhandled exceptions return structured `ApiResponse` with `code: 500`

## 6. Frontend Coding Standards

### 6.1 Component Structure

```jsx
// ✅ CORRECT — Named export, prop types documented
import { useState } from 'react';
import { fetchStudents } from '@/shared/services/studentService';

/**
 * StudentTable — Hiển thị danh sách học viên với phân trang
 * @param {Object} props
 * @param {string} props.classId - ID của lớp học cần lọc
 */
export function StudentTable({ classId }) {
  const [isLoading, setIsLoading] = useState(false);
  // ...
  return (
    <div className="overflow-x-auto rounded-lg border">
      {/* table content */}
    </div>
  );
}
```

**Rules:**
- Use **named exports** for all components (no default export unless it's a page/route component)
- Component file = one component per file (exception: small sub-components used only within the same file)
- Props MUST be documented with JSDoc for shared components
- No inline styles — use Tailwind utilities only

### 6.2 Custom Hooks

```js
// ✅ CORRECT — hooks/useStudents.js
import { useState, useEffect } from 'react';
import { studentService } from '@/shared/services/studentService';

export function useStudents(classId) {
  const [students, setStudents] = useState([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    if (!classId) return;
    setIsLoading(true);
    studentService.getByClassId(classId)
      .then(setStudents)
      .catch(setError)
      .finally(() => setIsLoading(false));
  }, [classId]);

  return { students, isLoading, error };
}
```

**Rules:**
- Custom hooks are placed in `shared/hooks/` (if reusable) or `features/[module]/hooks/` (if feature-specific)
- Hooks MUST return `{ data, isLoading, error }` pattern for data-fetching hooks
- Never call hooks conditionally (React rules of hooks)

### 6.3 API Service Layer

```js
// ✅ CORRECT — shared/services/studentService.js
import { apiClient } from './apiClient';

export const studentService = {
  getAll: () => apiClient.get('/students'),
  getById: (id) => apiClient.get(`/students/${id}`),
  create: (data) => apiClient.post('/students', data),
  update: (id, data) => apiClient.put(`/students/${id}`, data),
  delete: (id) => apiClient.delete(`/students/${id}`),
};
```

**Rules:**
- All API calls go through `apiClient` (Axios instance with JWT interceptors)
- Service functions MUST NOT use `try/catch` — let error bubble up to hook/component
- Base URL is injected from `import.meta.env.VITE_API_BASE_URL`

### 6.4 State Management

| Scope | Tool | Example |
|-------|------|---------|
| **Server state** | TanStack Query (React Query) | API data, pagination, caching |
| **Global app state** | Zustand store | Auth user, theme |
| **Local UI state** | `useState` / `useReducer` | Form state, modal open/close |
| **Form state** | React Hook Form + Zod | All form inputs |

**Rules:**
- Do NOT use Zustand for server state — use TanStack Query
- Do NOT put derived data into state — compute it from existing state/props
- Form validation schema (Zod) MUST be defined in the same file as the form or in `features/[module]/schemas/`

### 6.5 Routing Rules

```jsx
// app/routers/AppRouter.jsx
const AppRouter = () => (
  <Routes>
    <Route path="/" element={<PrivateRoute><MainLayout /></PrivateRoute>}>
      <Route index element={<DashboardPage />} />
      <Route path="students" element={<StudentsPage />} />
      <Route path="students/:id" element={<StudentDetailPage />} />
    </Route>
    <Route path="/login" element={<LoginPage />} />
  </Routes>
);
```

**Rules:**
- Route paths: all lowercase, kebab-case (`/student-enrollments`, NOT `/studentEnrollments`)
- Protected routes are wrapped with `<PrivateRoute>` HOC
- Page components are in `features/[module]/pages/` and named with `Page` suffix

## 7. Database & EF Core Conventions

### 7.1 Entity Naming

```csharp
// ✅ CORRECT
public class ClassSession        // Singular PascalCase noun
{
    public int Id { get; set; }
    public DateTime StartTime { get; set; }
    public DateTime EndTime { get; set; }
    public int TeacherId { get; set; }
    public Teacher Teacher { get; set; } = null!;
    public int RoomId { get; set; }
    public Room Room { get; set; } = null!;
}
```

### 7.2 EF Core Configuration (Fluent API)

```csharp
// DAL/Configurations/ClassSessionConfiguration.cs
public class ClassSessionConfiguration : IEntityTypeConfiguration<ClassSession>
{
    public void Configure(EntityTypeBuilder<ClassSession> builder)
    {
        builder.ToTable("class_sessions");              // snake_case table name
        builder.HasKey(x => x.Id);
        builder.Property(x => x.StartTime).IsRequired();
        builder.HasOne(x => x.Teacher)
               .WithMany(t => t.Sessions)
               .HasForeignKey(x => x.TeacherId)
               .OnDelete(DeleteBehavior.Restrict);
    }
}
```

**Rules:**
- Table names: `snake_case` (PostgreSQL convention)
- Column names: `snake_case`
- All entity configurations via Fluent API (NOT Data Annotations on entity classes)
- ALWAYS define `OnDelete` behavior explicitly — never rely on EF Core default cascade

### 7.3 Migration Rules

```bash
# Add migration
dotnet ef migrations add <MeaningfulName> --project CLS.DAL --startup-project CLS.Server

# Apply to database
dotnet ef database update --project CLS.DAL --startup-project CLS.Server
```

**Rules:**
- Migration names must be descriptive: `AddStudentPackageTable`, `AddRenewalAlertIndex`
- NEVER edit auto-generated migration files (create a new migration instead)
- Migrations are version-controlled — MUST be committed alongside the feature branch

## 8. Security Standards

| Area | Rule |
|------|------|
| **Authentication** | JWT Bearer via `Microsoft.AspNetCore.Authentication.JwtBearer` |
| **Password hashing** | BCrypt via ASP.NET Identity (NEVER store plain text) |
| **CORS** | Explicit allowed origins only — no wildcard `*` in production |
| **Sensitive config** | Use `appsettings.json` + User Secrets (dev) / Environment Variables (prod) |
| **Input validation** | FluentValidation on ALL incoming request DTOs |
| **SQL injection** | Use EF Core parameterized queries ONLY — no raw SQL string interpolation |
| **Role-based auth** | `[Authorize(Roles = "Admin,Teacher")]` on controller/action level |

## 9. Code Quality Rules

### 9.1 General

- Maximum method length: **30 lines** (extract helper methods if longer)
- Maximum class length: **300 lines** (split into partials or separate classes)
- No magic numbers/strings — use named constants
- Remove all `Console.WriteLine` debug lines before committing; use **Serilog** for logging

### 9.2 Git Commit Convention

Follow **Conventional Commits**:

```
feat(student): add renewal alert service
fix(auth): correct JWT expiry check
refactor(scheduler): extract conflict detection logic
docs(api): update student endpoint swagger comments
test(enrollment): add unit tests for package depletion calc
```

Format: `<type>(<scope>): <short description>`  
Types: `feat` | `fix` | `refactor` | `docs` | `test` | `chore` | `perf`

### 9.3 Branch Strategy

```
main          ← Production-ready, protected
develop       ← Integration branch
feature/      ← feature/cls-123-student-enrollment
fix/          ← fix/cls-456-renewal-alert-duplicate
```

## 10. Logging Standards (Serilog)

```csharp
// ✅ CORRECT — Structured logging
_logger.LogInformation("Student {StudentId} enrolled in class {ClassId}", studentId, classId);
_logger.LogWarning("Renewal alert for student {StudentId}: {DaysRemaining} days remaining", id, days);
_logger.LogError(ex, "Failed to send attendance email for session {SessionId}", sessionId);

// ❌ WRONG — String interpolation
_logger.LogInformation($"Student {studentId} enrolled"); // not structured
```

**Rules:**
- Use message templates (NOT string interpolation) for Serilog to capture structured properties
- Log levels: `Debug` (dev only), `Information` (user actions), `Warning` (soft failures), `Error` (exceptions)
- DO NOT log sensitive data (passwords, JWT tokens, PII without masking)

|-------|-----------|-------------|
| **Unit** | xUnit + Moq | Service methods, FluentValidation rules, utility functions |
